# 学浪案例配图替换指南

## 你需要怎么交付

1. 从 Figma 选中单个手机界面 Frame，使用 `PNG 2x` 导出。
2. 手机界面保持直角，不要自己加手机壳、圆角、阴影、背景或透视效果。
3. 导出后按下表的“你交付的 PNG 文件名”命名。这一列就是你要使用的文件名。
4. 你只需要提供 PNG；我会保留原图、转换 WebP、压缩体积，并在网页中统一处理圆角、裁切、层级、阴影和 PDF 版式。
5. 完整界面尽量保留顶部状态栏与底部操作区，不要贴边裁掉。

## 通用要求

- 优先从 Figma 直接选择产品界面 Frame 导出，不要导出带标题、策略说明、箭头和数据结论的整张汇报板。
- PNG 使用 2x 导出；界面本身保持原始比例，保持直角，不加手机模型、投影、圆角外框或背景装饰。
- 保留产品内真实文字。案例解释、设计判断、实验结论和中英文翻译由网页 HTML 承载。
- 文件使用下表中的固定文件名覆盖即可，MDX 文案不需要修改。覆盖后运行 `npm test -- tests/unit/xuelang-assets.test.ts`。
- 若新图尺寸变化，需要同步修改 `evidence/xuelang/manifest.json` 和 MDX 中对应的 `width`、`height`。

## 导出后的检查

1. 图片内没有案例标题、解释文案、方案字母或指标。
2. 顶部状态栏、底部操作区和关键内容没有被裁断。
3. 中文界面文字在网页 1440px 宽度下可直接辨认，不依赖 Lightbox。
4. Lightbox 放大后没有模糊、透明底黑边或相邻画板残留。
5. 中英文页面共用同一产品界面，翻译只存在于网页说明层。
