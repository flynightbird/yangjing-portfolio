# 学浪新旧版本擦除对比设计规格

## 目标

把“内容交付 → 连续学习体验”章节中现有的左右静态对比升级为可直接操作的擦除对比。交互需要强化版本演进叙事，同时保持产品界面可读、桌面端视觉优先，并在触控、键盘和 PDF 中提供完整信息。

## 已确认方向

采用 A「精准对比滑杆」。旧版位于左侧遮罩层，新版作为完整底图位于右侧。初始分界位置为画面宽度的 38%，让新版默认占据 62% 的主视觉。

不使用柔边擦除、鼠标跟随或滚动自动推进。准确比较产品界面比展示额外动效更重要，读者应能停在任意位置检查细节。

## 内容与素材

- 旧版素材：`old.jpg`，3324 × 2160。
- 新版素材：`new.jpg`，3324 × 2160。
- 两张图片具有相同尺寸和构图基准，直接叠放，不进行独立裁切或比例修改。
- 正式页面使用优化后的 WebP；原始 JPG 作为来源保留，不覆盖用户文件。
- 中文标签为“旧版 · 内容交付”和“新版 · 连续学习”。英文标签为“Before · Content delivery”和“After · Continuous learning”。

## 视觉结构

保留章节现有深色舞台与标题，静态双栏图片改为一张全宽对比画布：

```text
内容交付 → 连续学习体验
┌────────────────────────────────────────────┐
│ 旧版 · 内容交付       │    新版 · 连续学习 │
│                       ◉                    │
│        old.jpg         │      new.jpg       │
└────────────────────────────────────────────┘
  旧版说明                         新版说明
```

- 画布保持 `3324 / 2160` 的固定宽高比，桌面端占满章节内容宽度。
- 分界线使用高对比的 1px 浅色线；中心手柄为 48px 圆形控件，满足触控尺寸。
- 两侧标签置于画面顶部，不遮挡主要界面；使用半透明深色底和轻度背景模糊。
- 旧版、新版说明放在画布下方，避免把长文本压进图片。
- 不增加装饰性光效、渐变球或多余动效。擦除交互是本章节唯一视觉焦点。

## 交互

- 鼠标或触控按下画布任意位置即可移动分界线，并可持续拖动。
- 键盘聚焦后，左右方向键每次移动 3%；`Home` 和 `End` 分别移动到最小与最大位置。
- 分界位置限制在 4%–96%，避免标签和手柄离开画布。
- 手柄 hover、active 和 focus-visible 状态清晰，但不改变布局尺寸。
- 页面首次进入不自动播放，不抢夺滚动，也不记录读者上次位置。
- `prefers-reduced-motion` 下保留全部拖动能力，只移除非必要过渡。

## 组件边界

- 新建独立客户端组件负责位置状态、指针事件和键盘输入。
- `XuelangDarkComparison` 继续负责章节语义和内容传入，不把交互逻辑写入 MDX。
- 图片地址、替代文本、标签和说明继续由中英文 MDX 提供。
- 其他 `XuelangFigure`、Lightbox 和持续学习序列不受影响。

## 响应式与打印

- 桌面端保持全宽对比画布，优先确保产品界面清晰。
- 窄屏仍使用同一画布和比例，不为了手机端重排或削弱桌面构图；手柄和标签适当缩小，但触控区域不小于 44px。
- 打印和 PDF 不保留交互遮罩，改为等宽并排显示完整旧版与新版，并保留各自标签和说明。

## 无障碍

- 对比区域使用有名称的分组语义，滑杆暴露 `aria-valuemin`、`aria-valuemax` 与当前百分比。
- 两张图片保留各自准确的替代文本；视觉标签不作为替代文本的唯一来源。
- 焦点状态在深色背景上达到清晰可见的对比度。

## 验证标准

- 初始位置为 38%，画面默认呈现更多新版内容。
- 指针拖动、触控与键盘均可改变位置，范围始终在 4%–96%。
- 两张图片像素对齐，无跳动、拉伸、空白或水平溢出。
- 1440px 桌面视口中标题、标签、手柄和图片不重叠。
- 中文、英文、窄屏、reduced-motion 与打印布局均可用。
- 组件测试覆盖初始状态和键盘操作；端到端测试覆盖拖动与图片加载；PDF 验证确认两张完整图片均输出。
